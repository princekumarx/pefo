This workflow is provided by default along with Sumantra CRM package. This is a good example to depict how fields within same record can be updated when workflow is executed. We are set to do the following calculation.


Probability is equal to 40% when deal stage is screened.


1. Select Deal Pipeline in Select Module and my condition will be Until the first time the condition is true



2. The goal is to create events only when deal stage is screened; so, my condition will be Deal Stage is Screened. If condition is met, workflow will be triggered on the record. Once executed, workflow doesn't trigger action on that record again.



3. Click Save


4. Click Add Task and select Update Fields. Provide Task Title; this is only for your reference. Click Add Field button; as a result, two fields will be displayed. One is target field other is source field. Select Probability from the dropdown.



Clicking on source field brings up a popup. You can define calculations in this popup. In the popup, as we want to set the value to 40, I will suggest you select Raw Text from the first dropdown. Other options in dropdown are Field and Expression. Field is used to set the value to another field and on selection gives option to further select available fields from second dropdown. Expression is used for performing calculations and on selection gives option to further select available fields from second dropdown and functions from the third.


Type 40 from your keyboard in the text box. Click Save to update the value.



5. Click Save
